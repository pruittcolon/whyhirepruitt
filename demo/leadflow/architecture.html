Security Considerations

Understanding the demo vs. production security model

🔒 What's Protected

  • API keys use Origin Whitelisting (domain-locked)
  • No sensitive PII stored in localStorage for demo
  • Form submissions validate required fields client-side
  • Email addresses are controlled (hub receives all)

⚠ Demo Limitations

  • LocalStorage is per-browser (no cross-device sync)
  • No server-side validation or sanitization
  • Auth is mock-only (not production-ready)
  • Payment flow is simulated (no real transactions)
